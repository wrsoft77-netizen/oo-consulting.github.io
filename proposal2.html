웹 브라우저를 통해 리눅스 서버 운영 업무를 자동 수행하는 AI Agent 시스템 구축 제안입니다. 기존 정적 도구를 넘어, 강력하고 안전한 3단계 승인 프로세스를 통해 단순 조회뿐만 아니라 시스템 변경 작업까지 지원합니다.

Overview

무엇을 왜 만드는가?

사람이 직접 SSH에 접속하여 명령어를 실행하고 결과를 분석하던 업무 방식을 챗봇 인터페이스 기반의 AI Agent가 대체 및 대행합니다.

💬

자연어를 리눅스 명령으로

"메모리를 많이 쓰는 프로세스 상위 5개 보여줘"라고 질문하면 AI가 ps aux --sort=-%mem | head -6 형식으로 변환, 실행하고 설명해줍니다.

⚡

3단계 승인 기반의 확장성

읽기 전용 명령어는 자동 허용하고, 패키지 설치나 방화벽 변경과 같은 위험 명령어에는 단순 승인 및 암호 승인 프로세스를 거쳐서 어떠한 한계도 없이 서버를 제어할 수 있습니다.

📄

전문적인 보고서 작성 지원

복잡한 로그 데이터나 서버 상태를 AI가 종합하여 판단하고, 보안 이상 징후나 서버 부하 등을 알기 쉽게 요약한 보고서를 만들어 냅니다.

Architecture

전체 아키텍처 상세

브라우저에서 리눅스 커널까지 이어지는 단계적 통신 흐름.

1️⃣

웹 브라우저 (React)

사용자 입력 수집, WebSocket 통신, 실시간 응답 표시

2️⃣

FastAPI 백엔드

Tomcat을 통해 들어온 WebSocket 연결을 수신하고 AI Engine 중계

3️⃣

AI Agent 반복 루프 & 권한 분류

명령어 생성 ➔ 권한 분류 ➔ 승인 요청(웹 UI) ➔ 리눅스 실행 ➔ 결과 피드백 반복

실행 흐름 전체 예시

  1. WebSocket 질문: "최근 SSH 실패 분석해줘"
  2. LLM 프롬프트 전달 및 명령어 생성: journalctl -u sshd | grep Failed
  3. 분류 엔진에서 권한 레벨 판정 후 위험 수준에 따른 승인 대기
  4. 사용자 확인/암호 입력 인가 성공 시 리눅스 명령(subprocess) 실행
  5. 실행 결과값을 AI에게 재전송 및 심층 분석 수행
  6. AI 최종 답변 도출 (모든 과정은 감사 로그에 기록됨)
Prompt System

가장 안전한 의사결정: 프롬프트 원리

OS 운영 보조 역할을 위한 확실하고 체계적인 AI 규칙 사전 주입

🤖

시스템 프롬프트 (System Prompt) 정책

동작 원칙: 오직 <command> 형태로만 명령어를 도출합니다. 읽기 전용 조회 명령어는 자동 실행되며, 리눅스 서버 조작이나 시스템 변경에 해당하는 명령어는 관리자 승인을 획득한 이후에만 실행하여 안전을 보장합니다. 승인이 거부될 경우 적절한 대안을 제시하도록 지시합니다.

Permission Management

권한 관리: 3단계 승인 프로세스

✅

1. 자동 허용 (읽기 전용)

시스템 모니터링, 로그 및 네트워크 진단 등 조회 분석 명령은 관리자 승인 없이 즉시 자동 실행됩니다.

⚠️

2. 단순 승인 (중위험 명령어)

서비스/데몬 시작, 컨테이너 제어, 패키지 관리 등은 웹 다이얼로그의 클릭 승인 후에만 실행됩니다.

🚨

3. 암호 입력 승인 (고위험 명령어)

방화벽 설정, 사용자 계정, SELinux 등 서버 파괴나 보안 위험 요소는 관리자 비밀번호 검증 이후에 권한 부여됩니다.

명령어 카테고리 위험도 단계 승인 요건 및 대표 명령어
시스템 자원 점검, 로그, 네트워크 자동 허용 별도 승인 없음 (top, df, journalctl, ping)
서비스 데몬 관리, 프로세스, Docker 단순 승인 안내 + [승인] 클릭 (systemctl restart, kill, docker stop)
방화벽 통제, 디스크 조작, 계정, 서버 전원 암호 승인 PAM 관리자 암호 입력 필수 (firewall-cmd, useradd, reboot)
Command Matrix

주요 허용 명령어 (20대 핵심업무)

기존 블랙리스트 차단 방식에서 벗어나 유연하고 강력하게 서버 관리를 지원합니다.

✅ 자동 허용 업무

  • 모니터링: top, free -h, df -h, iostat
  • 분석 (로그, 네트웍): journalctl, tail, grep, ss, ping
  • 조회 및 모니터링을 포괄하는 읽기 전용 작업은 승인 절차 없이 스스로 병렬 실행됩니다.

⚠️ 단순 승인 허용 업무

  • 데몬 및 도커 제어: systemctl restart/stop, docker start/stop
  • 패키지/파일 제어: dnf install, kill, rm, chmod
  • 시스템 서비스 중단 또는 권한 변동 우려가 있는 중위험 커맨드군입니다.

🚨 암호 승인 허용 업무

  • 방화벽 및 인가 제어: firewall-cmd --add-port, passwd
  • 볼륨 제어 및 서버 제어: mkfs, mount, reboot, sysctl -w
  • 보안 및 서버 생애주기에 직결되는 고위험/민감 커맨드로, 암호 입력을 통해 철저하게 관리됩니다.
Advanced Functions

고급 기능 탑재 모델

🔁

Multi-Step (연쇄 실행)

"서버 전체 상태 점검해줘"라고 명령하면 uptime → free → df → docker ps 등 단계별 명령어를 단일 질의 위에서 스스로 반복하여 수행합니다.

📊

결과 기반 지능형 인사이트

결과값을 텍스트로만 뱉지 않으며, "x.x.x.x IP에서 23회 연속 접속 실패 감지됨, 방화벽 차단 권고 요망" 수준의 능동적 판단과 대안을 함께 제공합니다.
