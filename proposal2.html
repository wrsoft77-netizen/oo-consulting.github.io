웹 브라우저를 통해 리눅스 서버 운영 업무를 자동 수행하는 AI Agent 시스템 구축 제안입니다. 기존 정적 도구(v5)를 넘어 자유로운 자연어 명령과 다단계 분석을 지원합니다.

Overview

무엇을 왜 만드는가?

사람이 직접 SSH에 접속하여 명령어를 실행하고 결과를 분석하던 업무 방식을 챗봇 인터페이스 기반의 AI Agent가 대체 및 대행합니다.

💬

자연어를 리눅스 명령으로

"메모리를 많이 쓰는 프로세스 상위 5개 보여줘"라고 질문하면 AI가 ps aux --sort=-%mem | head -6 형식으로 변환, 실행하고 설명해줍니다.

⚡

기존 v5 대비 놀라운 확장성

기존 시스템이 16개 고정 도구만 활용했던 반면, 새로운 AI Agent는 읽기 전용으로 한정된 수백 가지의 리눅스 명령어를 창의적으로 조합하고 연달아 실행할 수 있습니다.

📄

전문적인 보고서 작성 지원

복잡한 로그 데이터나 서버 상태를 AI가 종합하여 판단하고, 보안 이상 징후나 서버 부하 등을 알기 쉽게 요약한 보고서를 만들어 냅니다.

Architecture

전체 아키텍처 상세

브라우저에서 리눅스 커널까지 이어지는 단계적 통신 흐름.

1️⃣

웹 브라우저 (React)

사용자 입력 수집, WebSocket 통신, 실시간 응답 표시

2️⃣

FastAPI 백엔드

Tomcat을 통해 들어온 WebSocket 연결을 수신하고 AI Engine 중계

3️⃣

AI Agent 반복 루프

명령어 생성 ➔ 안전 필터 ➔ 리눅스 실행 ➔ 결과 피드백을 반복 수행

실행 흐름 전체 예시

  1. WebSocket 질문: "최근 SSH 실패 분석해줘"
  2. LLM 프롬프트 전달 및 명령어 생성: journalctl -u sshd | grep Failed
  3. 샌드박스 검증 통과 후 리눅스 명령(subprocess) 실행
  4. 출력 결과를 AI에게 재전송
  5. AI 추가명령 (IP 집계) 생성 ➔ 실행 ➔ 최종 답변 도출
Prompt System

가장 안전한 의사결정: 프롬프트 원리

OS 운영 보조 역할을 위한 확실하고 체계적인 AI 규칙 사전 주입

🤖

시스템 프롬프트 (System Prompt) 정책

동작 원칙: 오직 <command> 형태로만 명령어를 도출하고, "읽기 전용" 지식을 사용토록 강제합니다. 파일 수정/삭제 및 시스템 변경은 원천 금지되며, 한 번에 하나의 명령어만 실행한 뒤 분석 결과를 기반으로 후속 조치를 진행하도록 지시합니다.

Sandbox Security

안전 설계: 3단계 방어선

🛡️

1. AI 프롬프트 지시 (Layer 1)

LLM 자체에 "시스템 변경 명령 절대 사용 불가" 사전인지 부여.

🚧

2. 코드 기반 안전 필터 (Layer 2) - 가장 강력함

절대 실행 불가(rm, chmod 등)와 위험 패턴(>, | bash)을 파이썬 코드로 검사 및 강제 차단.

🔒

3. OS 권한 제어 (Layer 3)

명령이 컨테이너 내부나 승인된 Non-Root 계정으로만 전송되어 시스템 파괴 불가 구조 확립.

명령어 카테고리 차단 항목 차단 논리
파일 조작/권한 rm, mv, touch, chmod, chown 무단 변경 및 중요 데이터 유실 우려 차단
네트워크/시스템 제어 shutdown, iptables, dnf, rpm 서버 무장애 환경 파괴 및 소프트웨어 무단 설치 방지
위험 기호/패턴 >, >>, | bash, -delete 덮어쓰기나 치환 등 은밀한 권한 우회 시도 원천 억제
Allowed Range

동작 가능한 명령어 (읽기 전용)

AI Agent가 업무 지원을 위해 자유롭게 사용할 수 있는 명령어 범주

🖥️ 시스템 / 하드웨어 자원 점검

  • uname -a, uptime, lscpu, free -h, lsblk
  • 단순 시스템 스펙 확인 및 CPU, Mem 스냅샷 조회

🌐 네트워크 / 프로세스 / 디스크 추적

  • ps aux, top, df -h, ss -tlnp, ping, dig
  • 사용 프로세스와 포트 현황 및 디스크 잔여량 모니터링

🔒 보안 / 접속자 환경 점검

  • who, w, last, id, faillock
  • 접속 중 사용자 이력, 로그인 실패 기록 탐지로 보안 사고 방지

🐳 도커 / 시스템 로그 심층 분석

  • docker ps, docker logs, journalctl, dmesg
  • 컨테이너 상태 점검, 에러 로그 취합을 통해 서비스 장애 진단
Advanced Functions

고급 기능 탑재 모델

🔁

Multi-Step (연쇄 실행)

"서버 전체 상태 점검해줘"라고 명령하면 uptime → free → df → docker ps 등 단계별 명령어를 단일 질의 위에서 스스로 반복하여 수행합니다.

📊

결과 기반 지능형 인사이트

결과값을 텍스트로만 뱉지 않으며, "x.x.x.x IP에서 23회 연속 접속 실패 감지됨, 방화벽 차단 권고 요망" 수준의 능동적 판단과 대안을 함께 제공합니다.
